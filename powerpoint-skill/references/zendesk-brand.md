## Color Principles

### Purpose
- Use **red** for danger/error
- Use **green** for success
- Use **yellow** for warnings
- Use **blue** for interactive elements and primary actions
- Limit status colors to avoid clutter

### Hierarchy
- **Primary Blue** (#2694d6) for interactive elements (buttons, links)
- **Neutral Greys** for non-interactive content
- Use color + text/icons to emphasize hierarchy

### Accessibility Requirements
- **Text**: Minimum 4.5:1 contrast ratio (WCAG AA)
- **Graphics/Icons**: Minimum 3:1 contrast ratio
- **Shade 600**: Meets 3:1 contrast against white
- **Shade 700**: Meets 4.5:1 contrast against white
- **Never use color alone** - always supplement with text or icons

## Typography

### Type Scale

Garden uses a responsive type scale:

- **XXXL** - Display headlines, cover slides
- **XXL** - Section headers, major points
- **XL** - Slide titles
- **LG** - Subheadings
- **MD** - Body text (default)
- **SM** - Captions, footnotes

### Font Modifiers

- **Bold** - Use for emphasis and headings
- **Monospace** - Use for code, technical data

### Typography Guidelines for Presentations

1. **Slide Titles**: Use XXL or XL size, Bold
2. **Body Content**: Use MD or LG size
3. **Bullet Points**: Use MD size
4. **Captions/Sources**: Use SM size
5. **Maximum text per slide**: Keep minimal - presentations should be visual
6. **Line spacing**: Use generous spacing for readability

## Presentation-Specific Guidelines

### Slide Layouts

1. **Title Slide**
   - Large title (XXXL or XXL, Bold)
   - Subtitle (LG or XL)
   - Background: White or Grey 100
   - Accent: Blue 600 or brand secondary color

2. **Content Slides**
   - Title: XXL, Bold, Blue 700
   - Body: MD, Grey 700
   - Background: White
   - Accent colors for callouts

3. **Data/Chart Slides**
   - Use primary colors for main data
   - Use secondary colors for supporting data
   - Ensure sufficient contrast
   - Include legends with clear labels

4. **Closing/Thank You Slide**
   - Similar to title slide styling
   - Include contact information (SM size)

### Best Practices

1. **Consistency**: Use the same color for the same type of content throughout
2. **White Space**: Don't overcrowd slides
3. **Contrast**: Ensure text is always readable
4. **Hierarchy**: Use color and size to guide attention
5. **Accessibility**: Include alt text for images, use high-contrast colors

### Common Presentation Types

#### Sales Decks
- **Primary Colors**: Blue 600, Green 600
- **Focus**: ROI, customer success stories
- **Charts**: Use green for growth, blue for stability

#### Product Demos
- **Primary Colors**: Blue 700, Purple 600
- **Focus**: Features, capabilities
- **Screenshots**: Frame with grey borders

#### Executive Briefings
- **Primary Colors**: Blue 700, Grey 700
- **Focus**: High-level metrics, strategic direction
- **Minimal text**: Focus on key numbers and charts

#### Customer Case Studies
- **Primary Colors**: Green 600, Blue 600
- **Focus**: Before/after, testimonials, results
- **Quotes**: Use blockquote styling

#### Business Cases/ROI
- **Primary Colors**: Green 600, Teal 600
- **Focus**: Financial data, projections
- **Charts**: Line charts for trends, bar charts for comparisons

## File Format Requirements

- **Resolution**: 1920x1080 (16:9) for standard presentations
- **Fonts**: System-safe fonts (Helvetica, Arial) or embed custom fonts
- **File Size**: Keep images optimized (< 5MB total per presentation)
- **Compatibility**: Save as .pptx for Google Slides compatibility
